CRITICAL SECURITY ISSUES (🔴 BLOCKING)
================================================================================

1. NO USER ISOLATION - SEVERITY: CRITICAL
   Files: backend/main.py (all endpoints)
   Problem: All endpoints open to all users
   Impact: User A can see/edit/delete User B's projects
   Example: GET /api/projects returns ALL projects (not filtered)
   Fix: Add `user: User = Depends(get_current_user)` + filter by user_id

2. NO USAGE TRACKING - SEVERITY: BUSINESS CRITICAL  
   Files: backend/services/usage.py + backend/main.py
   Problem: Usage service exists but never called
   Impact: Tier limits don't work, can't bill Pro users
   Fix: Integrate check_generation_allowed() and record_* functions

3. DATABASE/FILESYSTEM MISMATCH - SEVERITY: DATA CRITICAL
   Files: backend/main.py:59 + db/models.py
   Problem: Projects saved to filesystem, DB never written
   Impact: ProjectRecord table empty, multi-tenant impossible
   Fix: Create ProjectRecord row in DB when project created

4. NO TOKEN TRACKING - SEVERITY: BUSINESS CRITICAL
   Files: backend/services/llm.py + orchestrator.py
   Problem: Token usage never collected
   Impact: Can't bill Pro users accurately
   Fix: Return token counts from generate(), track in orchestrator

5. FRONTEND MISSING AUTH HEADERS - SEVERITY: AUTH CRITICAL
   Files: frontend/src/api/client.ts
   Problem: No JWT in Authorization header
   Impact: All requests fail if auth_enabled=True
   Fix: Add localStorage token + inject Bearer header

================================================================================
CRITICAL BUGS (🔴 BLOCKING)
================================================================================

6. USER REGISTRATION NOT COMMITTED
   File: backend/auth/routes.py:62
   Issue: await db.flush() but no await db.commit()
   Impact: User created but not persisted if transaction rolls back

7. NO ALEMBIC MIGRATIONS
   Missing: /backend/alembic/ directory entirely
   Impact: Cannot manage schema changes in production

8. API MISMATCH - reviseScene
   File: frontend/src/api/client.ts:35 vs backend/main.py:163
   Issue: Frontend sends 3 params, backend expects 4 (missing instructions)
   Impact: Revise endpoint always fails

9. STATUS ENUM MISMATCH
   File: Multiple locations
   Issue: Backend uses 'completed', frontend checks for 'done' AND 'completed'
   Impact: Generation completion might not redirect properly

10. REVISION FILENAME PARSING BROKEN
    File: backend/services/storage.py:290
    Issue: Split on "_" breaks if source contains underscores
    Example: "20260430_125030_before_manual_edit.md" parses wrong
    Impact: Revision history corrupted for sources with underscores
