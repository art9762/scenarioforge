### Перезагрузка страницы ломает прогресс генерации [CRITICAL BUG]
- [ ] `Generation.tsx`: при перезагрузке страницы `started=false` → показывает экран выбора глубины вместо прогресса
- **Что происходит:** бэкенд продолжает генерацию, но фронтенд об этом не знает. Пользователь видит кнопку "Начать генерацию" и может запустить **второй** параллельный пайплайн на тот же проект → двойная трата токенов, гонка записи в один файл, мусор в сценарии
- **Фикс:** на маунте `Generation.tsx` дёргать `GET /status` — если статус `generating`, сразу ставить `started=true` и запускать поллинг. Если `done` — редирект на сценарий. Если `error`/`stopped` — показать сообщение
- **Дополнительно:** бэкенд `/generate` должен возвращать 409 если генерация уже идёт, чтобы второй запуск был невозможен
